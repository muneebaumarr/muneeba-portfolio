AI Social Media Approval & Scheduling

Turns one piece of content into ready-to-publish posts for Instagram, Twitter, LinkedIn, and Medium — then routes everything through manager approval before it ever goes live.

4
platforms, one input
1
Slack click to approve
The problem

Repurposing a single piece of content for four platforms meant rewriting it four times, emailing it to a manager, waiting for feedback, and manually scheduling each version — every single post.

The system

A new record in Airtable kicks off an AI rewrite for each platform, a one-click Slack approval (approve / request edit / reject), and automatic scheduling through Buffer — with Airtable as the single source of truth throughout.

How it's built — 4 connected workflows
01

AI Repurpose Content

Watches for new content in Airtable, then splits into four branches that each call OpenAI to rewrite the piece in the right voice and format for Instagram, Twitter, LinkedIn, and Medium — and writes every version back to Airtable.

Airtable: new record → Router (4 platforms) → OpenAI rewrite × 4 → Airtable: save drafts
02

Social Media Approval & Scheduling

Picks up records that are ready for review, pulls the full draft from Airtable, and sends it straight into Slack as an approval request — no email, no copy-pasting between tools.

Airtable: watch records → Airtable: get record → Slack: send for approval
03

Slack Button Handler

Listens for the manager's click — Approve, Request Edit, or Reject — and routes the response automatically: approved posts get marked ready, edit requests go back to the team, and rejections are logged and closed out, each with a Slack confirmation.

Webhook: button click → Router (3 outcomes) → Airtable update → Slack confirmation
04

Buffer Scheduling

Once a post is approved, this workflow picks it up, creates the scheduled post directly in Buffer, and marks the Airtable record as scheduled — closing the loop with zero manual posting.

Airtable: watch approved → Buffer: create post → Airtable: mark scheduled

AI News Summarizer

Pulls the latest AI news every day from multiple sources, summarizes it with an AI model, and delivers a clean digest straight to your inbox — no manual searching, reading, or copy-pasting required.

2
news sources merged
Daily
fully automatic
The problem

Staying current on AI news means checking multiple sites every day, reading through articles, and mentally compiling the highlights — time that adds up fast and is easy to let slide.

The system

A daily trigger pulls the most recent articles from two news APIs, merges them, hands them to an AI model to summarize, and sends the finished digest as an email — ready the moment you check your inbox.

How it's built — 1 end-to-end workflow
01

Get the most recent articles

A daily schedule trigger fires two parallel requests — one to each news API — pulls back the freshest articles, and cleans up the fields before merging both sources into a single list.

Schedule trigger → HTTP Request ×2 (news APIs) → Edit Fields ×2 → Merge
02

Summarize and send to email

The merged articles are handed to an AI agent, which reads through everything and writes a short, digestible summary — then sends it directly as an email, with no extra step from the user.

AI agent (Groq) → Send message: Email

AI Content Creation Engine

Watches a content calendar in Google Sheets, researches the topic on the web, and generates platform-ready posts for LinkedIn, Facebook, and a full blog article — all from a single row.

3
content types, one trigger
0
manual research needed
The problem

Producing on-brand content for LinkedIn, Facebook, and a blog from scratch means researching the topic, writing three different versions in three different tones, and manually updating a tracker to know what's been done.

The system

Adding a topic to a Google Sheet triggers a live web search for current information, then three dedicated AI agents write a LinkedIn post, a Facebook post, and a full blog article in parallel — with the sheet automatically updated once it's done.

How it's built — 1 end-to-end workflow
01

Capture and research the topic

A new row added to Google Sheets kicks things off. The topic is cleaned up and sent out to a live web search, then split and aggregated so every downstream agent has fresh, accurate context to write from — not just whatever the model already "knows."

Google Sheets: row added → Edit Fields → Web search (Tavily) → Split & Aggregate
02

Write for LinkedIn, Facebook, and the blog — in parallel

Three separate AI agents, each with their own memory and tone, turn the researched topic into a LinkedIn post, a Facebook post, and a complete blog article at the same time — so the brand voice stays consistent without three separate writing sessions.

LinkedIn agent Facebook agent Blog agent
03

Save everything back to the tracker

All three finished pieces are written straight back into the same Google Sheets row they came from — so the content calendar always shows exactly what's ready to publish, with nothing scattered across docs or chat threads.

Google Sheets: update row
